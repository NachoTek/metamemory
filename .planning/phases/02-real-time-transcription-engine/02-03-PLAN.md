    Create confidence scoring module with normalization and color mapping:
    
    normalize_confidence(avg_log_prob: float) -> int:
    - Whisper log_probs range: -1.0 (high confidence) to -3.0 (low)
    - Map [-3.0, -1.0] to [30, 95] linear interpolation
    - Clamp: values > -1.0 return 95, values < -3.0 return 30
    - Return integer 0-100
    
    ConfidenceLevel enum:
    - HIGH = 80-100% (green)
    - MEDIUM = 70-80% (yellow)
    - LOW = 50-70% (orange)
    - VERY_LOW = 0-50% (red)
    
    get_confidence_level(confidence: int) -> ConfidenceLevel
    get_confidence_color(confidence: int) -> str - Return hex color code
    
    VISUAL DISTORTION EFFECT (for TRAN-04):
    - get_distortion_intensity(confidence: int) -> float (0.0 to 1.0)
    - 85%+ confidence: 0.0 (no effect)
    - 0% confidence: 1.0 (max effect, but capped for readability)
    - Linear interpolation between 85% and 0%
    - Max effect must keep text readable (cap at level that doesn't obscure)
    
    COLOR CODES:
    - Green (80-100%): #4CAF50
    - Yellow (70-80%): #FFC107
    - Orange (50-70%): #FF9800
    - Red (0-50%): #F44336
    
    Reference RESEARCH.md confidence extraction section (line 200-202) and 02-CONTEXT.md confidence display requirements.
  </action>

  <name>Task 2: Build hardware detection with psutil</name>
  <files>src/metamemory/hardware/detector.py, src/metamemory/hardware/__init__.py</files>
  <action>
    Create hardware detection module using psutil:
    
    SystemSpecs dataclass:
    - total_ram_gb: float
    - available_ram_gb: float
    - cpu_count_logical: int
    - cpu_count_physical: int
    - cpu_freq_mhz: Optional[float]
    - is_64bit: bool
    - platform: str
    
    HardwareDetector class:
    - detect() -> SystemSpecs - Gather all hardware info
    - has_minimum_requirements(specs: SystemSpecs) -> bool - Check if system can run dual-mode
    - get_warning_message(specs: SystemSpecs) -> Optional[str] - Return warning if below minimum
    
    MINIMUM REQUIREMENTS (for CFG-06 warning):
    - Minimum for single-mode (realtime only): 4GB RAM, 2 cores
    - Minimum for dual-mode (realtime + enhancement): 8GB RAM, 4 cores
    
    DETECTION LOGIC:
    - RAM: psutil.virtual_memory().total / (1024**3)
    - CPU: psutil.cpu_count(logical=True) for logical cores
    - Also detect physical cores: psutil.cpu_count(logical=False)
    - CPU frequency: psutil.cpu_freq().current (if available)
    
    PLATFORM HANDLING:
    - Windows: Use standard psutil (tested in Phase 1)
    - macOS/Linux: Same psutil API works cross-platform
    - For Phase 2, focus on Windows but keep portable
    
    CACHING:
    - Cache detection results for 60 seconds to avoid repeated system calls
    - Provide refresh() method to force re-detection
    
    Reference RESEARCH.md hardware auto-detection section (line 413-428).
  </action>
  <verify>python -c "from metamemory.hardware.detector import HardwareDetector; d = HardwareDetector(); specs = d.detect(); print(f'RAM: {specs.total_ram_gb:.1f}GB, CPUs: {specs.cpu_count_logical}'); assert specs.total_ram_gb > 0; assert specs.cpu_count_logical > 0; print('Hardware detection OK')"</verify>
