  <name>Task 2: Build JSON persistence with versioning</name>
  <files>src/metamemory/config/persistence.py</files>
  <action>
    Create SettingsPersistence class for atomic JSON file operations:
    
    - get_config_path() -> Path - Return path to config.json in app directory
    - load() -> Dict - Load and parse JSON config, handle missing file
    - save(settings: AppSettings) - Atomic write (write to temp, then rename)
    - migrate_config(config_dict: Dict, from_version: int) -> Dict - Handle version upgrades
    - get_default_settings() -> AppSettings - Return fresh default settings
    
    IMPLEMENTATION DETAILS:
    - Config location: Use app data directory (platform-appropriate)
      - Windows: %APPDATA%/metamemory/config.json
      - Or fallback to project directory for development
    - Atomic writes: Write to config.json.tmp, then os.rename() to config.json
    - This prevents corruption if crash during write
    - Handle FileNotFoundError gracefully - return default settings
    - Handle JSON decode errors - log warning, return defaults
    
    MIGRATION STRATEGY (for future-proofing):
    - Store config_version in JSON
    - If loaded version < current version, run migration chain
    - Example: version 1 -> 2 might rename a field
    - Keep migrations simple and additive (don't delete user data)
    
    SMART DEFAULTS:
    - Load existing config, compare to defaults
    - For any field not in saved config, use default value
    - This allows adding new settings without breaking old configs
  </action>
