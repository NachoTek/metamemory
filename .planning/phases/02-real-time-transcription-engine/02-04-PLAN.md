  <name>Task 1: Create RealTimeTranscriptionProcessor orchestration class</name>
  <files>src/metamemory/transcription/streaming_pipeline.py</files>
  <action>
    Create RealTimeTranscriptionProcessor that orchestrates all transcription components:
    
    - __init__(config: TranscriptionConfig)
      - Creates AudioRingBuffer
      - Creates VADChunkingProcessor (min_chunk_size from config)
      - Creates WhisperTranscriptionEngine (loads model based on config)
      - Creates LocalAgreementBuffer
      - Creates thread-safe result queue
    
    - start() - Start processing thread
    - stop() - Stop processing, wait for thread
    - feed_audio(chunk: np.ndarray) - Call from audio capture callback
    - get_results() -> List[TranscriptionSegment] - Get new segments (non-blocking)
    - is_model_loaded() -> bool - Check if Whisper ready
    - load_model(callback=None) - Async model loading with progress callback
    
    PROCESSING LOOP (runs in background thread):
    1. Check if enough audio in buffer (VAD processor decides)
    2. Extract chunk from buffer
    3. Run Whisper inference (this is the slow part - can take 0.5-2s)
    4. Get transcription with confidence scores
    5. Pass through local agreement buffer
    6. Put committed results in output queue
    7. Trim committed audio from ring buffer
    8. Sleep briefly to prevent CPU spinning
    
    THREADING MODEL:
    - Audio capture runs in AudioSession's consumer thread
    - Transcription inference runs in separate processor thread
    - feed_audio() is called from capture thread, must be thread-safe
    - get_results() called from main/UI thread
    - Use locks for buffer access
    
    LATENCY TARGET:
    - With base model: 0.5-1.5s inference time
    - With 1.0s min chunk: ~1.0-2.5s total latency
    - With tiny model: <1s inference for faster systems
    
    Reference RESEARCH.md Real-Time Streaming Loop pattern (line 328-387).
  </action>

    Create TranscriptStore for in-memory transcript with rich metadata:
    
    Word dataclass:
    - text: str

    - end_time: float
    - confidence: int (0-100)
    - is_enhanced: bool (for Phase 3)

    Segment dataclass:
    - words: List[Word]
    - start_time: float
    - end_time: float
    - avg_confidence: int
    
    TranscriptStore class:
    - add_words(words: List[Word]) - Add new words from transcription
    - get_segments(since_time: float = 0) -> List[Segment] - Get segments after time
    - get_all_words() -> List[Word] - Get all words in order
    - get_recent_words(count: int) -> List[Word] - Get last N words
    - clear() - Reset for new recording
    - to_markdown() -> str - Export for Phase 3/5
    - to_dict() -> dict - Serialize for saving
    
    WORD-BY-WORD DISPLAY:
    - Store individual words with timestamps
    - UI can render words as they arrive (streaming effect)
    - Words retain confidence for color coding
    - Supports future enhancement (mark as enhanced, bold display)
    
    CONCURRENCY:
    - Thread-safe for add_words from transcription thread
    - get_segments from UI thread
    - Use threading.Lock for word list access
    
    MEMORY MANAGEMENT:
    - For long recordings (30+ min), could be thousands of words
    - Each word is small (~100 bytes), 30 min = ~5000 words = ~500KB
    - Acceptable for in-memory storage
    - Trim old words if memory becomes concern (future optimization)
  </action>
